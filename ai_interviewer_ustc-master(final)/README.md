## 一、✨ 项目简介：解决什么问题？

**AI 面试官 USTC** 是一款基于大语言模型（LLM）与语音交互技术的智能化模拟面试系统。它旨在为求职者、在校学生及职场人士提供沉浸式、个性化、高仿真的面试训练体验，帮助用户在真实面试前提升表达能力、应变能力与专业知识水平。

**痛点解决**：  
传统面试训练存在“缺乏真实感”、“反馈不专业”、“资源有限”等问题。本项目通过 AI 技术模拟多角色面试官，提供语音对话、知识库检索、智能评估等一体化解决方案，让用户在任何时间、任何地点都能进行高质量面试训练。

## 二、💡 业务价值说明

### 对用户的实际意义：

- **提升面试信心**：通过反复模拟，降低真实面试紧张感。
- **专业能力评估**：AI 根据对话内容生成结构化评价报告，指出优势与不足。
- **个性化训练**：支持技术、算法、系统设计、行为面试等多种面试类型，适应不同岗位需求。
- **语音交互真实感**：支持语音输入与语音播报，模拟真实面试场景，提升表达能力。

### 对教育/培训机构的潜在价值：

- 可作为高校就业指导、IT 培训机构的辅助教学工具。
- 降低教师/教练一对一辅导的时间成本，提升培训效率。
- 可扩展为多领域面试题库，适用于不同专业背景的学生。

## 三、🚀 AI 创新性说明

### 1. **🤖 AI 作为核心面试官**

- 基于大语言模型（如 Qwen、StepFun）构建多角色面试官，具备专业领域知识与人设。
- 支持上下文记忆与连贯追问，模拟真实面试中的深度互动。

### 2. **🎤 语音交互与智能断句**

- 集成 StepFun ASR/TTS，实现语音输入与语音输出。
- 创新性设计**智能断句模块**（`chunking_tool`），支持流式 TTS 与实时语音播报，提升对话流畅度。

### 3. **🔍 检索增强生成（RAG）**

- 构建多领域向量知识库（如计算机科学），面试过程中实时检索相关知识点。
- 检索结果作为背景知识注入 LLM，提升回答的专业性与准确性。

### 4. **📊 智能评估与报告生成**

- 面试结束后，调用 Qwen-max（思考模式）对对话进行深度分析。
- 生成结构化评价报告，涵盖技术能力、问题解决、沟通表达等多个维度。

## 四、🛠️ 技术实现说明

### ⚠️ 关于 API 密钥配置的重要说明

本项目在 `config.py` 文件中采用**硬编码方式配置 API 密钥**，主要基于以下考虑：

#### 🎯 为了方便评委会快速配置环境

- 考虑到项目评审环境需要快速搭建和配置
- 避免评委在环境变量配置上花费额外时间
- 确保评审过程能顺利测试所有核心功能

#### 如果您要部署到生产环境：

1. **立即修改** `config.py` 中的 API 密钥
2. **删除**已经提交到版本控制的历史密钥
3. **撤销**已在代码中暴露的密钥权限
4. **采用**安全配置方式

#### 📝 当前配置仅为演示目的

```python
# config.py（当前配置，仅用于评审演示）
STEPFUN_API_KEY = "sk-xxxxxxxxxxxx"  # 演示用密钥，部署前请更换！
DASHSCOPE_API_KEY = "sk-xxxxxxxxxxxx"  # 演示用密钥，部署前请更换！
```

### **安全提示**：保护您的 API 密钥就像保护您的银行卡密码一样重要！🚨

### 整体架构：

```
前端（Streamlit） → 语音/文字输入 → 后端（Python 模块化处理） → AI 服务（LLM + RAG + TTS/ASR）
```

### 核心模块：

- **`llm_agent.py`**：大语言模型对话引擎，支持流式输出。
- **`rag_engine.py`**：基于 Chroma + DashScope Embeddings 的检索增强模块。
- **`audio_processor.py`**：语音处理模块，集成 ASR/TTS 与智能断句。
- **`ai_report.py`**：面试报告生成模块，调用深度推理模型进行评价。
- **`app_streamlit.py`**：响应式 Web 界面，支持多 Tab 交互。

### 关键技术栈：

- **Python 3.8+**：主开发语言。
- **Streamlit**：快速构建交互式 Web 应用。
- **LangChain + Chroma**：构建 RAG 系统与向量检索。
- **DashScope API / StepFun API**：提供 LLM、嵌入模型、语音服务。
- **异步处理**：录音、转写、语音合成全流程异步化，提升响应速度。

## 五、🎨 交互与设计说明

### 关键界面与流程：

1. **侧边栏集中控制**：
   
   - 选择面试官类型（技术、算法、系统设计、行为、计算机基础、自定义）。
   - 开启/关闭语音播报与知识库检索。
   - 一键开始新对话，清理历史记录。

2. **四 Tab 主界面**：
   
   - **🎙️ 语音对话**：点击麦克风录音，自动识别并发送，AI 回复语音播放。
   - **💬 文字对话**：传统聊天界面，支持文字输入与流式显示。
   - **📚 RAG 知识检索**：展示每轮对话检索到的知识片段与来源。
   - **📊 面试报告**：下载对话记录，生成并下载 AI 评价报告。

3. **专业会客厅风格**：
   
   - 浅灰/米白背景、深灰正文、深蓝强调色。
   - 卡片式对话气泡，左右区分用户与 AI。
   - 响应式布局，支持移动端与桌面端。

## 七、🚀 开始使用

### 环境准备：

- Python 3.8+，推荐使用 Conda 管理环境。
- 有效的 StepFun API 密钥（用于语音服务）。
- 可选的 DashScope API 密钥（用于 RAG 功能）。

### 快速启动：

```bash
git clone <repository-url>
cd ai_interviewer_ustc
conda create -n AI_interviewer python=3.8
conda activate AI_interviewer
pip install -r requirements.txt
streamlit run app_streamlit.py
```
